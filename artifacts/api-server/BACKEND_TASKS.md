# Задачи по доработке Backend (Dream Cars Studio)

## 1. Конфигурация и Окружение
- [ ] **Environment Variables**: Настроить использование `.env` файла.
  - Добавить переменные: `PORT`, `DATABASE_URL`, `CORS_ORIGIN`.
  - Убедиться, что `.env` добавлен в `.gitignore`.
- [ ] **Logging**: Внедрить структурированное логирование (Winston или Pino) для отслеживания запросов и ошибок.

## 2. База Данных (Drizzle ORM + PostgreSQL)
- [ ] **Подключение**: Настроить соединение с базой данных в `src/lib/db.ts` (или аналогичном).
- [ ] **Схема Бронирования (`bookings`)**:
  - `id`: uuid/serial
  - `name`: string
  - `phone`: string
  - `service_type`: string (enum)
  - `message`: text (optional)
  - `status`: enum ('pending', 'confirmed', 'completed', 'cancelled')
  - `created_at`: timestamp
- [ ] **Схема Услуг (Optional)**: Таблица `services` для управления ценами и описанием через админку.
- [ ] **Migrations**: Настроить и проверить работу миграций Drizzle (`drizzle-kit`).

## 3. API Маршруты и Контроллеры
- [ ] **Бронирование (`/api/bookings`)**:
  - `POST /`: Создание новой заявки. Сохранение в БД.
  - Валидация входных данных (Zod: имя обязательно, телефон обязательно, валидный формат).
- [ ] **Контактная форма (`/api/contact`)**:
  - `POST /`: Обработка формы "Связаться с нами".
- [ ] **Услуги (`/api/services`)** (если нужно динамически):
  - `GET /`: Получение списка активных услуг и цен.

## 4. Уведомления (Notification Service)
- [ ] **Telegram Bot / Email**:
  - При создании заявки отправлять уведомление администратору.
  - Интеграция с Telegram API или Email провайдером (Resend, Nodemailer).

## 5. Middleware и Безопасность
- [ ] **Валидация**: Middleware для валидации запросов с использованием Zod схем.
- [ ] **Error Handling**: Глобальный обработчик ошибок (Global Error Handler) для перехвата исключений и отправки понятных ответов клиенту.
- [ ] **Rate Limiting**: Ограничение количества запросов (express-rate-limit) для защиты от спама.
- [ ] **CORS**: Проверить настройки CORS для разрешения запросов только с доверенных доменов (frontend).

## 6. Администрирование (Optional)
- [ ] **Auth**: JWT аутентификация для админ-панели.
- [ ] **Admin Routes**:
  - `GET /api/admin/bookings`: Просмотр всех заявок.
  - `PATCH /api/admin/bookings/:id`: Изменение статуса заявки.

## Текущее состояние (Audit)
- Проект использует `express`, `cors`, `drizzle-orm` (по зависимости).
- Структура каталогов (`routes`, `middlewares`, `lib`) создана, но пуста или содержит только health-check.
- Отсутствует реальная логика работы с БД и бизнес-логика.
